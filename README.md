# Software-Test-Automation
# To ensure that my code and software are both functional and secure, I focus on input validation, error handling, and thorough testing. In my project, I made sure that all fields (such as contact ID, phone number, and names) were validated before being accepted, which helps prevent invalid or harmful data from entering the system . I also used unit testing to confirm that each method works as expected and handles errors properly. Writing tests for both successful cases and failure cases helped me verify that the program behaves reliably and securely.

#To interpret user needs, I focus on understanding the requirements and translating them into clear features. For example, users needed to be able to add, update, and delete contacts, so I designed methods in the service class to handle each of those actions . I also made sure to include constraints, like limiting field lengths and enforcing unique IDs, because these reflect real-world expectations for a contact management system. Breaking down user needs into smaller tasks made it easier to implement and test each feature.

When designing software, I take a structured and step-by-step approach. I start by identifying the main components (such as the Contact class and the ContactService class), then define their responsibilities. After that, I implement the core functionality and follow up with testing to ensure everything works correctly. I also think about maintainability by keeping the code organized and easy to update. This approach helps me build software that is not only functional but also easy to improve and expand in the future.
